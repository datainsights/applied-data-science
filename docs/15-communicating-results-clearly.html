Communicating Results Clearly

Published

Mar 2026

  • ID: DS-L15
  • Type: Premium
  • Audience: Intermediate to Advanced
  • Theme: Translating correct interpretation into clear communication

In the previous lesson, we learned how to interpret models responsibly.

Now we take the next step:

How do we communicate those interpretations clearly to others?

This is where many analyses fail.

Not because the model is wrong,
but because the communication is misleading, vague, or overstated.

Why Communication Matters

Model results are rarely consumed by the person who built them.

They are read by:

  • collaborators
  • decision-makers
  • clinicians or domain experts
  • stakeholders without technical background

This means:

clarity is not optional — it is part of the analysis.

The Communication Gap

A model produces outputs such as:

  • coefficients
  • metrics (MAE, R²)
  • predictions

But stakeholders need:

  • meaning
  • context
  • limitations

Bridging this gap is the goal of this lesson.

From Technical Output to Clear Statement

Consider this model result:

R² = 0.48
MAE = 44.3

Poor Communication

  • “The model performs reasonably well”
  • “The results are acceptable”

These are vague and uninformative.

Better Communication

  • “The model explains about 48% of the variation in disease progression”
  • “Predictions are off by about 44 units on average”

This translates metrics into understandable meaning.

Explaining Model Behavior

Instead of listing coefficients:

bmi coefficient = 529

Poor Communication

  • “BMI has a coefficient of 529”

Better Communication

  • “Higher BMI values are associated with higher predicted disease progression”
  • “BMI is one of the strongest predictors used by the model”

Communicating Uncertainty

All models have uncertainty.

Ignoring it leads to overconfidence.

Instead of saying:

  • “The model is accurate”

Say:

  • “The model shows moderate predictive ability”
  • “Performance is consistent across multiple data splits”

Avoiding Common Mistakes

1. Overgeneralization

❌ “This applies to all patients”
✔ “This applies to this dataset and similar conditions”

2. Causal Language

❌ “BMI causes progression”
✔ “BMI is associated with predicted progression”

3. Overconfidence

❌ “The model is highly reliable”
✔ “The model is reasonably stable but has limitations”

Structuring a Clear Report

A good result summary should include:

1. Objective

  • What problem was addressed?

2. Data Context

  • What dataset was used?

3. Model Summary

  • What model was applied?

4. Key Results

  • Metrics and main patterns

5. Interpretation

  • What the results mean

6. Limitations

  • What the model does not tell us

Example: Clear Communication

Objective:
Predict disease progression using clinical features.

Results:
The model explains about 48% of the variation in progression (R² ≈ 0.48).
Predictions differ from actual values by about 44 units on average.

Interpretation:
BMI and related features are strong predictors in this dataset.

Limitations:
These results reflect associations within the dataset and do not establish causation.

CDI Insight

Good analysis is not complete until it is clearly understood.

What This Means in Practice

When communicating results:

Focus on:

  • clarity over complexity
  • meaning over metrics
  • honesty over confidence

Avoid:

  • vague language
  • unsupported claims
  • unnecessary technical detail

Key Insight

Clear communication does not simplify the analysis.

It makes the reasoning visible and trustworthy.

Summary

  • Models produce technical outputs
  • Communication translates outputs into meaning
  • Claims must remain aligned with evidence
  • Clarity improves trust and usability
  • Good communication is part of good analysis
