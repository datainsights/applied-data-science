Limitations and Responsible Use

Published

Mar 2026

  • ID: DS-L18
  • Type: Premium
  • Audience: Intermediate to Advanced
  • Theme: Understanding the limits of models and responsible use of results

In the previous lesson, we completed an end-to-end workflow:

  • data
  • modeling
  • evaluation
  • interpretation
  • communication
  • decision context

Now we step back and ask a critical question:

What are the limits of everything we just did?

This is where analytical maturity develops.

Why This Matters

Our model showed:

  • moderate predictive ability
  • reasonable stability
  • interpretable feature influence

It is tempting to trust these results.

But every model has limits.

Understanding those limits is part of responsible analysis.

Dataset Limitations

A model only learns from the data it is given.

This means:

  • it cannot detect patterns that are not present
  • it may reflect biases in the dataset
  • it may not represent broader populations

For example:

  • the diabetes dataset represents a specific cohort
  • it does not include all possible clinical variables
  • it may not reflect different populations or settings

CDI Insight

Models inherit the strengths and weaknesses of their data.

Model Limitations

Models simplify reality.

For example:

  • linear models assume linear relationships
  • tree-based models depend on observed splits

These simplifications can:

  • miss complex relationships
  • exaggerate certain patterns
  • behave differently under new conditions

Even a well-performing model is still an approximation.

Generalization

A model trained on one dataset:

  • may not perform the same on new data
  • may fail when conditions change

Good performance here does not guarantee performance elsewhere.

This is especially important in:

  • healthcare
  • finance
  • real-world decision systems

CDI Insight

Good performance on one dataset is not proof of universal validity.

Confounding and Hidden Structure

Some relationships appear strong because:

  • variables are correlated
  • important variables are missing
  • underlying mechanisms are not observed

For example:

  • BMI may appear important
  • but it may reflect other biological or behavioral factors

This is why:

association is not causation

Uncertainty in Predictions

All predictions contain error.

We observed this through:

  • MAE values
  • residual plots
  • variation across cross-validation folds

This means:

  • individual predictions may be inaccurate
  • model outputs should not be treated as exact values

CDI Insight

Models reduce uncertainty. They do not eliminate it.

Sensitivity to Data and Design Choices

Model results depend on:

  • feature selection
  • preprocessing steps
  • model type
  • parameter choices

Changing any of these can lead to different results.

This reinforces:

  • the importance of reproducibility
  • the need for careful interpretation

Responsible Use

Using models responsibly means:

  • acknowledging limitations
  • avoiding overgeneralization
  • avoiding causal claims
  • communicating uncertainty clearly

Practical Questions to Ask

Before using a model in practice, ask:

  • What data was this model trained on?
  • Where might this model fail?
  • What assumptions does it rely on?
  • What are the consequences of incorrect predictions?

These questions help prevent misuse.

Example: Responsible vs Irresponsible Use

Irresponsible

  • “This model predicts disease progression accurately for all patients”
  • “BMI is the main cause of progression”

Responsible

  • “This model shows moderate predictive ability within this dataset”
  • “BMI is an important predictor in this model, but not necessarily causal”

Key Insight

A useful model is not one that is perfect.

It is one that is:

  • understood
  • appropriately applied
  • used with awareness of its limits

Connecting Back to the Workflow

Everything we built remains valid:

  • pipelines improve reproducibility
  • cross-validation improves stability assessment
  • interpretation helps explain model behavior

But all of these operate within limits.

Recognizing those limits is what turns:

analysis → responsible analysis

Summary

  • All models have limitations
  • Data defines what models can learn
  • Generalization is not guaranteed
  • Associations are not causation
  • Uncertainty is always present
  • Responsible use is part of analysis
